# Основы диалектов языков

## Уровни (слои)

Языки Metalang всегда охватывают три уровня спецификаций:

* _**Абстрактный**_
* _**Семантический**_
* _**Прикладной (Исполнительный)**_.

_**Прикладной слой**_ предназначен для машиночитаемых спецификаций трансляторов прикладных языков программирования, таких как C, C#, Java, Rust, Python и многих других.

Машиночитаемые спецификации трансляторов языков **Metalang**, в свою очередь, написаны на языках спецификаций _**Семантического слоя**_. В семантическом слое находятся наиболее обширное подмножество спецификаций. Там находятся спецификации для:

* **Базовых элементов языков**\
  семантика операторов, переменных, исключений, примитивов синхронизации, грамматики, модулей и др.
* **Платформо-зависимого оборудования**\
  IoT, микропроцессоры, ПК, консоли, мобильные платформы, серверное оборудование и др.
* **Предметной области и бизнеса**\
  Медицины, промышленности, финансов, логистики, банкинга и др.

Семантический слой описывает лишь взаимосвязь сущностей и их ограничения, допустимые инварианты, а также протоколы их взаимодействия. Спецификации семантического слоя описаны на машиночитаемом языке.\
\
Сущности семантического слоя описываются на языках спецификаций _**Абстрактного слоя**_. Абстрактный слой описывает математический аппарат, позволяющий наиболее гибким образом описать абстрактное вычислительное устройство и топологическую карту исполнения, которые позволяют описать все семантики на универсальном языке, который подвергается статическому анализу (но не выполнению), а также проверке гипотез о функциональной эквивалентности сущностей и процедур (см. Оптимизации). Также абстрактный слой содержит механизмы версионирования, которые позволяют дифференцировать и изменять сущности более низкоуровневых слоёв спецификации единым образом, сохраняя при этом, возможность возврата к предыдущей версии и предоставляя базовые механизмы метапрограммирования.\
\
Абстрактный слой - это единственный слой, который содержит спецификации самого себя. Он не может быть машиночитаемым по определению и предназначен для людей, однако, языки этого уровня достаточно тривиальны для ручной реализации на любом целевом оборудовании для последующей **раскрутки**.\
\
Только прикладной слой содержит спецификации трансляторов языков, предназначенных для исполнения. Поэтому прикладной слой может содержать только контекстно-свободные грамматики по иерархии Хомского. Языки Абстрактного и Семантического слоя **не предназначены для непосредственного выполнения** на действительном оборудовании. Они предназначены для статического анализа и проверки гипотез оптимизации и генерации кода, поэтому на них не накладывается ограничение по сложности выполнения (эффективная вычислимость), что позволяет частично использовать контекстно-зависимые грамматики, например, представление сущностей языка в рекурсивной форме. Это одна из **важнейших** особенностей трансляторов семейства Metalang.

## Раскрутка (Bootstraping)

Типичный цикл внедрения транслятора, совместимого с Metalang специфичен для железа, на котором будет производиться трансляция, но так или иначе является формой раскрутки компилятора. Это обусловлено принципом обратной совместимости.<br>

Именно раскрутка позволяет гарантировать корректную реализацию базовых возможностей языка, поскольку требует корректной реализации системы контроля версий (версионирования), которая предоставляют фундаментальные возможности для репрезентации конструкций языка на том же языке, а также обеспечивает сквозное преобразование сущностей между уровнями спецификаций. Это обеспечивает расширенные возможности для метапрограммирования, что является фундаментом для наиболее гибкого и хирургически точного вмешательства в процесс трансляции.<br>

Раскрутка также позволяет собрать транслятор из сущностей абстрактного слоя, что приводит к однородному представлению базовых элементов транслятора языка. Это способ формализации и унификации семантической составляющей транслятора, благодаря которому, становятся доступны оптимизации на уровне семантики языка (и нескольких языков), а также их бесшовное взаимодействие (передача и обработка данных, ограничения контрактов, вызов процедур между языками, использование общих абстракций).
